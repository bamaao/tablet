在 Android 平板上，**盘点（Stocktake/Audit）** 是最重头的功能。它不能像普通的出入库那样简单，因为药库环境复杂，药剂师可能两手抓药，无法频繁点击屏幕。

因此，盘点 UI 的设计核心是：**“盲操优先、语音驱动、差异高亮”**。

---

### 2. 核心交互流程：语音 + 手动

#### **场景：盘点散装中药**

1. **激活**：用户点击“开始盘点”或说“开始盘点当归”。
2. **视觉反馈**：
* 中心区 B 弹出大字体显示：“**当归**”。
* 单位自动锁定在**最小单位 (g)**。


3. **语音输入**：用户报数：“当归，实盘五百克”。
* **STT 解析**：`{ item: "当归", quantity: 500, unit: "g" }`。
* **UI 自动填充**：500g 自动填入输入框，声波条闪烁绿色。


4. **智能拆包逻辑集成**：
* 如果用户说：“盘点当归 3 包另 200 克”。
* 系统自动计算：。
* UI 同时显示：`3包 + 200g`。

### 3. 特色 UI 组件设计

#### **A. 差异指示器 (Difference Indicator)**

当输入实盘数后，右侧 C 区动态更新：

* **绿色 (+)**：盘盈。提示：“是否入库多余药品？”
* **红色 (-)**：盘亏。**强制弹出**原因选择器（如：过期丢弃、正常损耗、漏记）。

#### **B. “盲盘”语音反馈模式**

针对不方便看屏幕的情况，开启“语音播报”：

* 系统：“当前盘点：当归。系统库存 5 包。请报数。”
* 用户：“四包。”
* 系统：“收到，实盘 4 包，亏损 1 包。下一项：黄芪。”

### 4. React Native 核心组件逻辑 (TypeScript)

```tsx
// 盘点状态模型
interface AuditState {
  systemQty: number;      // 账面数量 (最小单位)
  actualQty: number;      // 实盘数量 (最小单位)
  discrepancy: number;    // 差异值
  reason?: string;        // 盈亏原因
}

// 语音指令解析后的回调处理
const handleVoiceAudit = (transcript: string) => {
  const result = parseAuditCommand(transcript); // 解析：当归 450克
  if (result.matched) {
    setActualQty(result.val);
    // 自动触发 TTS 反馈
    TTS.speak(`当归实盘${result.val}${result.unit}, 差异${result.diff}克`);
  }
}

```

---

### 5. 异常处理原型

* **药品未找到**：如果语音报了一个不在列表里的药，UI 底部弹出半透明浮窗：“未找到该药品，是否新增临时盘点项？”
* **单位冲突**：用户对西药（盒）报了“克”，系统语音纠错：“阿莫西林仅支持‘盒’或‘板’，请重新报数。”
