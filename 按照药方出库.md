在医药库存管理中，“按照药方出库”是一个高级功能，它涉及到**多项药品联动、处方模板管理以及自动单位换算**。对于中药来说，还需要支持“付数”（即同一个方子抓几副药）的逻辑。

以下是针对该功能的详细逻辑设计及 UI 原型说明：

---

### 1. 业务逻辑设计：处方出库 (Prescription Outbound)

处方出库的核心是将一个“药方”看作一个**药品集合**，通过输入“付数”倍增所有药品的消耗量。

* **逻辑流**：选择药方 -> 设定付数（如 7 副） -> 系统自动计算总需求（单味药量 × 付数） -> 检查每味药的库存 -> 自动拆包（如库存不足整包，自动从散装扣除） -> 确认出库。
* **拆包联动**：如果药方中某味药需要 200g，而散装库只有 50g，系统应自动提示或执行“拆 1 包（500g）”的操作，确保流程不中断。

---

### 2. UI 交互原型 (平板端设计)

#### **左侧：药方选择与配置**

* **搜索框**：支持语音搜索药方名称（如：“搜索补中益气汤”）。
* **常用药方列表**：显示名称及包含的药味数。
* **付数调节器**：醒目的大数字选择器（[ - ] **7 付** [ + ]）。

#### **右侧：出库明细预览 (关键区)**

* **清单表格**：
| 药品名称 | 单剂量 | 总需求量 | 当前库存 | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| 当归 | 15g | 105g | 500g | ✅ 充足 |
| 黄芪 | 30g | 210g | 50g | ⚠️ 需拆 1 包 |
| 连翘 | 10g | 70g | 20g | ❌ 缺货 |
* **操作按钮**：[一键拆包备货] [确认批量出库]。

#### **语音交互支持**

* “按**补中益气汤**抓**七付**。” -> 系统自动填充药方并设置付数为 7。
* “当归加 5 克。” -> 针对当前方子进行临时调剂。

### 4. 核心执行代码 (TypeScript)

利用 WatermelonDB 的事务处理，确保药方中所有药品的扣减同时成功或失败。

```typescript
/**
 * 执行处方出库事务
 * @param items 药方明细数组
 * @param dosageCount 付数
 */
export const executePrescriptionOutbound = async (items: any[], dosageCount: number) => {
  await database.write(async () => {
    const ops = items.map(async (item) => {
      const totalNeeded = new Big(item.singleDose).times(dosageCount).toNumber();
      const stock = await getStockRecord(item.medicineId);

      // 1. 更新库存余额 (优先扣除散装 base_qty)
      const updateOp = stock.prepareUpdate(s => {
        s.baseQty = new Big(s.baseQty).minus(totalNeeded).toNumber();
        s.lastUpdated = Date.now();
      });

      // 2. 创建变动流水
      const transOp = database.get('transactions').prepareCreate(t => {
        t.medicineId = item.medicineId;
        t.type = 'OUT';
        t.quantity = -totalNeeded;
        t.unit = item.unit;
        t.note = `处方出库: ${dosageCount}付`;
        t.createdAt = Date.now();
      });

      return [updateOp, transOp];
    });

    const allOps = (await Promise.all(ops)).flat();
    await database.batch(...allOps);
  });
};

```
